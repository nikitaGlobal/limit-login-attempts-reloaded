Изменения от e33b42d до HEAD (плагин limit-login-attempts-reloaded). Базовый коммит e33b42d — Github workflows phpcs updated.

Вкратце: добавили полноценную 2FA и rescue links, интеграцию с api.limitloginattempts.com, поток входа с кодом по email, поправили админские уведомления и кучу всего по безопасности и UX. 36 файлов, примерно +4248 строк.

Вкладка 2FA в админке. Новая вкладка в меню плагина (tab-mfa.php). Настройки по ролям — кто может включить 2FA. Генерация rescue codes, можно скачать PDF, копировать, печатать. При включении чекбокса «Включить 2FA» вылезает попап с резервными ссылками; при активации сабмитим форму, при закрытии попапа перезагружаем страницу. Приватность-нотис перенесли под блок включения 2FA, лишние описания убрали. Важность сохранения rescue-ссылок подсветили предупреждающим стилем. В попапе единый стиль заголовков, кнопки PDF/Copy/Print в одну строку. Если нет OpenSSL — показываем предупреждение на вкладке MFA. PDF-либы грузим только на вкладке MFA и по требованию, jsPDF взяли локально вместо CDN.

Rescue links. Одноразовые ссылки с TTL для восстановления доступа если потерял 2FA. Храним в транзиентах (префикс LLA_MFA_TRANSIENT_RESCUE_PREFIX), можно сделать неограниченный TTL через конфиг. Использование ссылки атомарное, чтобы не было double-spend. Глобальный cooldown между вызовами rescue-эндпоинта (одна штука за LLA_MFA_RESCUE_USE_COOLDOWN секунд). Вместо WP Cron для временного отключения MFA — транзиенты; почистили CODE_LENGTH, убрали pending_links, после успешного rescue 2FA снова включается сама. В очистке транзиентов везде один префикс. После редиректа по rescue показываем нейтральное сообщение без деталей. Попап с кодами только когда MFA включён. UX попапа: без иконки закрытия, кнопки Печать и Подтвердить и закрыть, стабильная вёрстка модалки, печать без лишнего; PDF поправили, html2canvas убрали. Тексты про rescue расширили, в т.ч. пример с технической проблемой.

MFA flow. Полный поток с провайдерами и callback после входа. MfaProviderRegistry, провайдер email LlarMfaProvider по MfaProviderInterface. SessionStore — сессии в транзиентах, один секрет из handshake для verify и send_code, send-code перевели на POST, секрет в URL не таскаем. CallbackHandler обрабатывает llar_mfa=1&token=...&code=..., проверяет сессию и OTP, дергает API verify, логинит юзера и редиректит. REST send-code с AJAX-fallback, эндпоинт в mfa-flow, пофиксили callback send-code, убрали debug. MfaRestApi, MfaFlowSendCode, MfaApiClient — клиент к api.limitloginattempts.com, rate limit на handshake. send_email_secret храним для send-code. Порядок HTTP-транспортов поправили: fopen, потом WP, потом curl. Усилили валидацию и защиту по send-code и callback. Со вкладки 2FA убрали отладочный UI и статистику MFA flow.

Безопасность. Пофиксили SQL-инъекции, rescue-коды в БД шифруем. Усилили генерацию и проверку rescue (криптостойко, константное сравнение против timing attack). XSS в выводе rescue-ссылок поправили, rescue-эндпоинт вынесли в отдельный handler. Timing attack при верификации rescue и в других местах закрыли. В wp_die и при ошибках — неинформативные сообщения, чтобы не светить внутреннее состояние. CDN заменили на локальные либы (в т.ч. jsPDF). SSL/HTTPS для MFA требуем, но можно разрешить без HTTPS через единые нотисы в админке; «HTTPS рекомендуется» только на вкладке Debug. Единые причины блокировки MFA через константы LLA_MFA_BLOCK_REASON_* (ssl, salt, openssl), детерминированная соль для rate-limit.

Архитектура. MfaController разнесли по классам с DI: MfaManager, MfaSettings, MfaValidator, MfaBackupCodes, RescueCode, MfaEndpoint плюс интерфейсы. Все MFA-методы из LimitLoginAttempts в MfaController и дальше по классам. MfaConstants — фасад для LLA_MFA_* (длина кода, кол-во, TTL, cooldown, префиксы транзиентов, лимиты PDF и т.д.). AdminNoticesController — один вывод нотисов на странице настроек (auto-update, https-recommended, flash), один шаблон views/notices.php. MfaRescueEndpointHandler перевели на Config, выровняли с MfaController. Меньше дублирования, интерфейсы и PHPDoc, почистили до пяти основных классов. Разбили God Object, улучшили DI, кэш ролей, валидация путей, единые проверки capability/SSL/ролей. error_log только при WP_DEBUG; get_rescue_url делегирован, переменная таблицы в БД, кэш OpenSSL.

Стили и фронт. Новый _mfa-tab.scss, правки в _settings и limit-login-attempts.scss, обновлён скомпилированный CSS. В JS заменили var на let/const. Сообщение про отключённый MFA вынесли в mfa-disabled-message.js. Пофиксили порядок: lla-main регистрируется до локализации MFA, приоритет enqueue для MFA 1000. llar_vars мержим, а не перезаписываем.
